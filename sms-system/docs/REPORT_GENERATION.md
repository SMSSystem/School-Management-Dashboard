## 1. Feature Specification

From [`PROJECT_SPEC_AND_ANALYSIS.md`](./PROJECT_SPEC_AND_ANALYSIS.md) ¬ß1.8:

> Reports are generated on demand. The system pulls every grade (`results`) for the student in the selected term, plus every feedback comment (`feedback_comments`) submitted by that student's teachers for the same term.

```
Teacher submits feedback
  ‚îî‚îÄ‚îÄ stored against (studentId + teacherId + classId + termId)

Institution admin or senior teacher generates report for a student+term
  ‚îî‚îÄ‚îÄ System pulls:
        ‚îú‚îÄ‚îÄ all results for that studentId in that termId (all classes, full term)
        ‚îî‚îÄ‚îÄ all feedback_comments for that studentId in that termId
              ‚îî‚îÄ‚îÄ Report viewable by: super_admin (all), institution_admin (institution),
                                      senior_teacher (dept), regular_teacher (class),
                                      student (own), parent (child's)
```

Reports are stored as **snapshots**. Generation reads current `results` and `feedback_comments`, computes the output, and writes it as a Firestore document. Subsequent views read the stored document and do not re-query live collections. If a grade is corrected after generation, the report must be re-generated by an eligible role.

## 2. Design Decisions

### 2.1 Role Access

**Decision:** `institution_admin` and `senior_teacher` can generate reports. All other roles are read-only.

| Action | super_admin | institution_admin | senior_teacher | regular_teacher | student | parent |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| Generate (write to `reports`) | ‚ùå | ‚úÖ institution | ‚úÖ dept scope | ‚ùå | ‚ùå | ‚ùå |
| View (read from `reports`) | ‚úÖ all | ‚úÖ institution | ‚úÖ dept scope | ‚úÖ class scope | ‚úÖ own only | ‚úÖ child's |

**Generate scope definitions:**

| Role | Can generate for‚Ä¶ |
|---|---|
| `institution_admin` | Any student in their institution |
| `senior_teacher` | Any student in any class within their department (`isSeniorTeacherFor`) |

> **Note ‚Äî Admin re-generation restored (2026-05-30):** An earlier design removed institution admin generate access; it has since been restored. `institution_admin` is now a first-class generator alongside `senior_teacher`.

---

### 2.2 Grades Data Model

**Decision:** The platform supports both flat and weighted grading systems, configurable per institution. Report generation branches on which system is active.

#### Flat mode

Average `score / maxScore` across all results for the student in the term.

```
results/{id}
  studentId ¬∑ teacherId ¬∑ classId ¬∑ termId ¬∑ institutionId ¬∑ departmentId
  assessmentName    // e.g. "Midterm Exam"
  score             // e.g. 78
  maxScore          // e.g. 100
  // weight absent ‚Üí flat mode
```

#### Weighted mode

Computes `Œ£(score / maxScore √ó weight)` across all results. Weights per student in the term should sum to 1.0 ‚Äî enforced at the application layer, not in Firestore rules.

```
results/{id}
  studentId ¬∑ teacherId ¬∑ classId ¬∑ termId ¬∑ institutionId ¬∑ departmentId
  assessmentName ¬∑ score ¬∑ maxScore
  weight            // 0‚Äì1 ‚Äî present ‚Üí weighted mode
```

#### Grading configuration

**Resolved (2026-05-31): Institution-level.** `gradingSystem: 'flat' | 'weighted'` lives on the `institutions/{id}` document. All classes in the institution share the same system. The grading config UI is a dropdown on the institution admin settings page (N-2). Report documents snapshot the `gradingSystem` value at generation time so retrospective views reflect the grading logic that was active when the report was produced.

---

### 2.3 UI Housing

**Decision:** A single dedicated `/reports` page, role-scoped.

- Admins see all reports across their institution.
- Students see only their own reports.
- Parents see their linked child's reports.
- Teachers (regular and senior) see reports for students within their scope.
- The generate button is visible only to `institution_admin` and `senior_teacher`.

## 3. Firestore Collections

### 3.1 `feedback_comments`

```
feedback_comments/{docId}   // auto-generated ID
  studentId      string     // links to users/{uid}
  teacherId      string     // links to users/{uid}
  classId        string     // links to classes/{classId}
  termId         string     // links to terms/{termId}
  institutionId  string     // multi-tenancy scoping
  departmentId   string     // required by isSeniorTeacherFor() rule
  comment        string     // written feedback text
  createdAt      string     // ISO 8601 timestamp
```

**Uniqueness:** One comment per teacher per student per class per term. `FeedbackCommentForm` enforces this with an upsert ‚Äî query for an existing document matching `studentId + teacherId + classId + termId`; if found, `updateDoc`; if not, `addDoc`.

---

### 3.2 `reports`

```
reports/{docId}              // auto-generated ID
  studentId        string    // links to users/{uid}
  studentName      string    // denormalized at generation time
  termId           string    // links to terms/{termId}
  termName         string    // denormalized at generation time
  institutionId    string    // multi-tenancy scoping
  institutionName  string    // denormalized at generation time
  generatedAt      string    // ISO 8601 timestamp
  generatedBy      string    // uid of the user who triggered generation
  generatedByRole  string    // role of the generator (audit trail)
  gradingSystem    string    // snapshot: 'flat' | 'weighted' ‚Äî locked at generation time
  departmentId     string?   // present for senior_teacher-generated; absent for institution_admin-generated
  grades           array     // snapshot of results records for this student+term
  feedback         array     // snapshot of feedback_comments, with teacherName denormalized
  overallScore     number    // computed at generation time
```

**Denormalization rationale:** `studentName`, `termName`, and `institutionName` are stored on the report document at generation time so PDF rendering requires no additional Firestore reads. `institutionName` is free ‚Äî the institution document is already fetched for `gradingSystem`. `teacherName` is denormalized into each feedback entry at generation time via parallel point reads (`getDoc(doc(db, 'users', teacherId))`).

**Upsert behaviour:** Before writing, `generateReport.ts` queries for an existing report matching `studentId + termId + institutionId`. If found, it calls `updateDoc` (re-generation). If not, it calls `addDoc`.

## 4. Firestore Security Rules

All three rule blocks are published to the Firebase Console as of 2026-05-31 and recorded in [`firebase-rules.md`](./firebase-rules.md).

### 4.1 `feedback_comments`

```
match /feedback_comments/{docId} {
  allow read: if (isTeacherOrAbove() && sameInstitution(resource.data.institutionId))
    || resource.data.studentId == request.auth.uid
    || (isParent() && exists(/databases/$(database)/documents/student_parents/$(request.auth.uid + '_' + resource.data.studentId)));

  allow create: if writingToMyInstitution()
    && (isAdminOrAbove()
      || isClassTeacherFor(request.resource.data.classId)
      || isSeniorTeacherFor(request.resource.data.departmentId));

  allow update: if sameInstitution(resource.data.institutionId)
    && (isAdminOrAbove()
      || (isTeacher() && resource.data.teacherId == request.auth.uid)
      || isSeniorTeacherFor(resource.data.departmentId))
    && institutionNotChanged();

  allow delete: if isAdminOrAbove() && sameInstitution(resource.data.institutionId);
}
```

---

### 4.2 `reports`

```
match /reports/{docId} {
  allow read: if (isTeacherOrAbove() && sameInstitution(resource.data.institutionId))
    || resource.data.studentId == request.auth.uid
    || (isParent() && exists(/databases/$(database)/documents/student_parents/$(request.auth.uid + '_' + resource.data.studentId)));

  allow create: if writingToMyInstitution()
    && (isAdmin()
      || isSeniorTeacherFor(request.resource.data.departmentId));

  allow update: if sameInstitution(resource.data.institutionId)
    && (isAdmin()
      || isSeniorTeacherFor(resource.data.departmentId))
    && institutionNotChanged();

  allow delete: if isAdminOrAbove() && sameInstitution(resource.data.institutionId);
}
```

> **`departmentId` on reports:** When a `senior_teacher` generates a report, the document must include `departmentId` so `isSeniorTeacherFor` can resolve correctly on re-generation. For `institution_admin`-generated reports, `departmentId` is absent ‚Äî the `isAdmin()` branch does not require it. Reports are term-wide; a `senior_teacher`-generated report includes grades and feedback from all of the student's classes, including those outside the teacher's department.

---

### 4.3 `institutions` (rule expansion)

**N-2b ‚Äî Expanded to allow `institution_admin` to update their own institution document** (required for the grading system settings UI). Published 2026-05-31.

```
match /institutions/{institutionId} {
  allow read: if isSignedIn();
  allow create: if isSuperAdmin();
  allow update: if isSuperAdmin()
    || (isAdmin() && myInstitutionId() == institutionId);
  allow delete: if isSuperAdmin();
}
```

## 6. PDF Export (A-5) ‚Äî Design Decisions

All A-5 items complete as of 2026-05-31. Issue #38 in [`ISSUES_AND_GAPS.md`](./ISSUES_AND_GAPS.md) resolved.

| Decision | Resolution |
|---|---|
| Library | `@react-pdf/renderer` |
| Visual style | Styled report card ‚Äî sky-blue header block, bordered grades table, section dividers, institution logo placeholder |
| Institution/student/term names in PDF | Denormalized onto `ReportDocument` at generation time ‚Äî zero extra reads on PDF open |
| Teacher attribution | One attribution line per feedback comment; teacher names denormalized at generation time via parallel `users/{teacherId}` point reads |
| Logo | Static bordered placeholder ("School Logo") ‚Äî no `logoUrl` field, no Firebase Storage dependency |
| Export modes | In-app preview (`PDFViewer`) + direct download (`PDFDownloadLink`), surfaced via a per-row "PDF" button opening a full-screen modal |
| Lazy loading | `PDFPreviewModal` and `ReportPDF` loaded via `React.lazy` ‚Äî the ~1.3 MB `@react-pdf/renderer` chunk is deferred until the user first clicks a PDF button |
| Double render | `PDFViewer` and `PDFDownloadLink` each render `<ReportPDF>` independently. Library-level limitation; not a bug. Not perceptible for a report of this size. |

**PDF sections rendered by `ReportPDF.tsx`:**

1. **Header** ‚Äî sky-blue background; institution name; "Academic Report" subtitle; student name; term name; generated date; logo placeholder
2. **Grades table** ‚Äî columns: Assessment ¬∑ Score ¬∑ Max ¬∑ %; Weight column present only when `gradingSystem === 'weighted'`; alternating row shading; bold column headers
3. **Teacher Comments** ‚Äî one block per feedback entry with right-aligned attribution (`teacherName ¬∑ createdAt`); "No teacher comments recorded." when `feedback.length === 0`
4. **Footer** ‚Äî overall score (`overallScore.toFixed(1)%`); grading system label; generated-by role

> **Note:** All PDF styling uses `StyleSheet.create()`. Tailwind classes have no effect inside `@react-pdf/renderer` components.

## 9. Known Limitations

| Limitation | Detail |
|---|---|
| Teacher name fallback in mock mode | Mock `feedback` entries have no `teacherName`. The PDF shows "Teacher" as a fallback. Live reports show real names. |
| Double PDF render per modal open | `PDFViewer` and `PDFDownloadLink` each render `<ReportPDF>` independently ‚Äî two renders per modal open. `@react-pdf/renderer` library limitation; not a bug. No perceptible performance impact for a report of this size. |
| `PDFViewer` on mobile | Renders an `<iframe>` with a blob URL. Some mobile browsers (particularly iOS Safari) block iframe blob URLs ‚Äî the preview may not display. The **Download PDF** button works as a fallback on all platforms. |
| Tailwind has no effect inside `ReportPDF` | `@react-pdf/renderer` components are not DOM elements. All PDF styling uses `StyleSheet.create()`. Changes to PDF visual style must be made there, not in Tailwind classes. |
| `overallScore = 0` when no results exist | If no `results` documents exist for the selected student+term+institution combination, the generated report will have `overallScore: 0`. The report still writes successfully. Re-generate after seeding results data. |
| `institutionName` empty when no institution document exists | `generateReport.ts` looks up `doc(db, 'institutions', institutionId)` by document ID. If no document exists at that path, `institutionName` falls back to `''` and `gradingSystem` falls back to `'flat'`. Seed an `institutions` document whose document ID matches the institution admin's `institutionId`. |
| `super_admin` sees no reports in live mode | The `onSnapshot` subscription on the reports page filters by `institutionId`. When `institutionId === '*'` (super admin), the subscription is skipped. Super admin live-mode report visibility is not yet implemented. |
| Live list pages for teachers/students/results still use mock data | Items 2c in [¬ß5.2](#52-deferred-items) ‚Äî deferred until Firestore is seeded. |
